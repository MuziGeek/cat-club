# /dev-blog - 发布开发笔记到博客

将本次开发会话的工作内容生成博客文章，发布到 `D:\GitProject\muzi-blog\source\_posts\note\project\cat-club` 目录。

## 使用方式

```bash
/dev-blog                          # 生成开发笔记博客
/dev-blog --title "自定义标题"      # 指定文章标题
/dev-blog --dry-run                # 仅预览，不写入文件
```

## 执行步骤

### Step 1: 确定文章序号

1. **列出现有文章**
   ```bash
   ls "D:\GitProject\muzi-blog\source\_posts\note\project\cat-club"
   ```

2. **提取最大序号**，新文章序号 = 最大序号 + 1
   - 现有文件：`cat-club-1.md`, `cat-club-2.md`, `cat-club-3.md`
   - 新文件：`cat-club-4.md`

### Step 2: 收集开发信息

分析以下内容：

1. **Git 变更**
   ```bash
   git status --short
   git diff --stat
   ```

2. **当前日期**
   ```bash
   date "+%Y-%m-%d %H:%M:%S"
   ```

3. **分析本次会话完成的功能**
   - 新增的文件
   - 修改的文件
   - 实现的功能
   - 遇到的问题及解决方案

### Step 3: 生成博客文章

## 文章风格指南

### 语言风格

- **第一人称**：使用"我"、"今天"等
- **轻松活泼**：可以使用表情符号，如 ✅ ⏸️ 🎨
- **技术准确**：代码示例要可运行
- **图文并茂**：善用表格、代码块、列表

### 标签选择规则

根据开发内容选择相关标签：

| 开发内容 | 推荐标签 |
|----------|----------|
| UI/组件开发 | Flutter, Widget, UI |
| 状态管理 | Riverpod, 状态管理 |
| 手势交互 | 手势交互, 拖拽 |
| 网络请求 | Firebase, API |
| 数据存储 | Firestore, 持久化 |
| 图片处理 | 图片上传, 裁剪 |
| 认证相关 | 认证, Firebase Auth |
| 动画效果 | 动画, Rive, Lottie |

### 内容深度

- 记录**为什么这样做**，不仅仅是做了什么
- 踩坑经历要详细，包含错误信息和解决过程
- 代码示例要有注释，解释关键部分

## 注意事项

1. **日期格式**：必须是 `YYYY-MM-DD HH:mm:ss` 格式
2. **Front Matter**：YAML 格式，注意缩进
3. **分类路径**：固定为 `[笔记, 项目, 宠物俱乐部]`
4. **代码高亮**：使用 ```dart 或其他语言标识
5. **中文标点**：使用中文标点符号
6. **链接检查**：确保内部链接有效
